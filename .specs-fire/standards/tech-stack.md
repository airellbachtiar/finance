# Tech Stack

## Overview

Family Ledger is a full-stack Next.js application. One codebase, one deployment target (Vercel), serverless Postgres via Neon. Chosen for low maintenance overhead — this is a personal/family project run by a single developer, not a team product.

## Core Technologies

### Language

**Primary**: TypeScript
**Version**: 5.x

### Framework

**Framework**: Next.js (App Router)
**Version**: 14.x or later
**Rationale**: Full-stack in one codebase (UI + API routes), first-class Vercel deployment, easy PWA upgrade path later.

### Runtime

**Runtime**: Node.js
**Version**: 20.x LTS

## Data Layer

### Database

**Type**: Relational
**Database**: PostgreSQL (hosted on Neon, free tier)
**Version**: 15+

### ORM / Data Access

**Tool**: Prisma
**Rationale**: Type-safe schema and queries, straightforward migrations, good fit for a small relational domain model (households, members, expenses, splits, settlements).

### Caching

Not configured. Not needed at this scale (~5 users).

## API Layer

### API Style

**Style**: Next.js Route Handlers (REST-ish, colocated with the app)

### Documentation

**Tool**: None formal — internal API only, no external consumers.

## Frontend

### UI Framework

**Framework**: Next.js (React) with server components where possible
**Version**: matches Next.js version above

### Styling

**Approach**: Tailwind CSS

### State Management

**Solution**: Server state via React Server Components + fetch; no global client state library needed at this scope.

## Infrastructure

### Hosting

**Platform**: Vercel (Hobby/free tier), custom domain purchased by the user and attached via DNS (CNAME/A record)

### Containerization

Not configured — Vercel's managed build/deploy pipeline is used directly.

### CI/CD

**Platform**: Vercel Git integration (auto-deploy on push to main; preview deployments on branches)

## Development Tools

### Package Manager

**Manager**: npm (or pnpm — decide at scaffold time)

### Build Tool

**Tool**: Next.js built-in build (Turbopack/Webpack, whichever Next.js defaults to at chosen version)

### Linting

**Linter**: ESLint (`eslint-config-next`)
**Config**: `.eslintrc.json`, Next.js defaults + TypeScript rules

### Formatting

**Formatter**: Prettier
**Config**: `.prettierrc`, default settings (2-space indent, single quotes)

## Dependencies

### Production Dependencies

- `next` — application framework
- `react`, `react-dom` — UI
- `@prisma/client` — database access
- `next-auth` — authentication (email + Google providers)
- `tailwindcss` — styling

### Development Dependencies

- `prisma` — schema/migrations CLI
- `typescript` — type checking
- `eslint`, `eslint-config-next` — linting
- `prettier` — formatting
