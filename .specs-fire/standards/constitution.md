# Project Constitution

> Universal policies that apply to ALL code in this project.
> This file is always inherited from root — modules cannot override it.

## Git Workflow

- **Commit Style**: Conventional commits (feat:, fix:, chore:, docs:, refactor:, test:)
- **Branch Strategy**: Direct commits to `main` — solo project, no PR ceremony. The FIRE builder commits and pushes at the end of each completed work item.
- **Main Branch**: main is always deployable, and every push auto-deploys via Vercel's git integration

## Code Review

- Solo project (single admin/developer) — the FIRE checkpoint process (plan approval, code review, test report) substitutes for PR review
- Security-sensitive or money-affecting changes (auth, balance engine, settlements) always go through Validate mode, never autopilot

## CI/CD

- All pushes to main auto-deploy to Vercel (production)
- Preview deployments on feature branches before merging to main

## Security Policies

- **No secrets in code** — use Vercel environment variables (`.env.local` for local dev, gitignored)
- No real bank credentials or payment execution — this app only *displays* settlement instructions (IBAN, amount, reference); it never moves money or talks to a bank API
- IBANs and other personal financial identifiers are visible only to the household members involved, never public
- Dependencies must be from trusted sources (npm registry, actively maintained packages)

## Documentation

- README kept up to date with local setup instructions (env vars needed, how to run migrations)
- Non-obvious domain decisions (e.g. why balances are derived, not stored) documented in code comments at the point of computation
